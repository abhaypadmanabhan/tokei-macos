## What ships

### Fixed
- **P0 — app no longer overheats the machine.** A CPU sampler on the running build put the
  entire idle burn in **SwiftUI ViewGraph rendering**, not parsing: the dashboard is a
  `Window` scene whose view graph is retained when the window closes, so per-second countdown
  `TimelineView`s + Swift Charts + a `repeatForever` status pulse kept re-rendering forever —
  ~20–24% CPU as a background menu-bar app with no window open. `DashboardView` now gates its
  whole content on `NSApplication.occlusionState` (`WindowVisibilityMonitor`): off screen it
  collapses to a static fill. **Measured: idle 0%, window-open ~11%.** (`df55434`, `cfda70f`)
- **P0 (secondary) — parser CPU on large actively-written logs.** Both JSONL parsers cache
  per-file aggregates + resume from byte offset instead of full-re-parsing every sync
  (`000f393` Claude, `#55` Codex). File-cache now evicts stale paths so it can't grow
  unbounded (`844de08`).
- **#49 — Cursor cooldown keyed by durable WorkOS account id** (survives token refresh; one
  account's 429 never blocks another; raw cookie never persisted). (`4b2a182`)

### Added
- **#51 — Plan-cost presets + best-effort Cursor plan detection.** Preset picker prefills the
  price; a badged, un-persisted "detected" suggestion never overwrites a manual entry or
  fabricates `$0`; other providers stubbed. (`001c304`)

### Documented
- **#54 — Gemini one-time CLI sign-in + token-refresh recovery** (README + provider-spec) and
  a matching provider-row empty-state hint. (`5777f51`, part of `001c304`)
