## Security review

`/security-review` run on the `main...dev` diff (2026-07-08). **No HIGH/MEDIUM findings** at the
≥7-confidence bar. Sink-by-sink verification: Cursor cookie sent only as a `Cookie` header to the
hardcoded `cursor.com` HTTPS host (`httpShouldHandleCookies=false`, standard TLS, no logging, no
persistence); JWT decode is `sub`-only, memory-safe, fail-safe to nil; `ps`/`lsof` use array args
(no shell); temp-copy SQLite reads use static SQL + READONLY + UUID temp dirs; protobuf/CSV/JSON
parsing is bounds-checked and memory-safe. Two items considered + dismissed: the localhost-only
TLS accommodation (`AntigravityQuotaClient`, loopback-only → not MITM-reachable) and the unverified
JWT `sub` (correct by design — self-auth to the user's own account). No triage actions required.
