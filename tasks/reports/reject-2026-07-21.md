## Verified root causes (read in code, 2026-07-21)

1. **Heatmap range — CONFIRMED BUG.** `DashboardViewModel.heatmap(for:)` calls
   `UsageAnalytics.heatmapMatrix(hourlyTotals:calendar:)` with **no range
   argument**. `hourlyTotals` is `[Date: Int]` (carries dates) but `heatmapMatrix`
   folds ALL of them into the 7×24 weekday×hour grid regardless of range. The
   top-bar range control drives `viewModel.range` (trend, delta, stats respond)
   but the heatmap never reads it. Fully fixable: add `range:`/`now:` to
   `heatmapMatrix` and filter dates before folding (mirror `filteredDailyTotals`).
2. **Quota lens shows tokens — CONFIRMED.** `OverviewAgentGrid` cells come from
   `OverviewView.agentModels`, which resolves stat as: today tokens → else quota%
   → else "—". It NEVER reads the `metric` (Usage/Quota) lens, so in Quota mode
   cells still show tokens. Quota mode must show used% (and % remaining to reset).
3. **Missing providers in quota — NEEDS RUNTIME TRACE.** The quota rows render
   only providers present in `viewModel.utilization` (`tightestByProvider`).
   Codex/Claude quota come from local session windows; Cursor/Antigravity via the
   live connector. If Claude live quota is enabled but no `Utilization` is
   produced (auth/error/window-shape), it silently drops. The fixing agent must
   trace on the real machine WHY Claude (and any other enabled provider) is absent
   from `utilization`, then surface it — do not guess.
4. **Menu-bar quota switch — POLISH.** `MenuBarView` already wraps the metric
   toggle in `withAnimation(PadzyMotion.quick)`; Quota mode shows a
   threshold-coloured fill bar + reset countdown. Add: bar-fill load animation
   (reduce-motion safe) and a pace marker (last-period usage as a red notch/arrow)
   so the user reads "ahead/behind vs before".
5. **Hover details — CONFIRMED MISSING.** `LineTrendChart` has a static peak
   annotation + cap RuleMark but NO interactive hover/selection. `ActivityHeatmap`
   has only `.help(model.tooltip)` (slow system tooltip; cells are weekday×hour,
   so "that day" is a category, not a calendar day). Add real hover: trend →
   nearest-point rule + styled callout (date · total · top agent); heatmap →
   immediate styled hover tooltip (weekday+hour · top agent · tokens), and decide
   whether a calendar-day heatmap variant is wanted for the "that day" reading.
