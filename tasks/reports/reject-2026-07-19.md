## Classification

NOT a functional failure — all 2026-07-19 wave acceptance criteria passed manual
QA. This is an information-architecture / density reject. `dev` stays healthy;
**no revert**. Fix-forward as a new UI work package.

## Root cause (design, not code)

Additive waves each claimed a sidebar destination (Overview, then Value, plus
7 static provider rows, Settings) with no consolidation pass. Sidebar ≈ 10 rows
at 230 pt width; provider rows are navigation for what is really *data*
(entities that belong IN the dashboard), and Value is a lens on the same
snapshot data as Overview — a tab, not a place.

## Direction (agreed with Mobbin references)

- Kill the 230 pt sidebar. Top bar: in-content tab pills OVERVIEW / VALUE
  (StackAI pattern), Settings via icon (top-right), reclaim full width.
- Providers = horizontal chip strip at top of the dashboard (glyph + one live
  stat per chip) with trailing `+` chip → Add Agent (LangSmith chip-row
  pattern). Chip click → provider drill-in with back affordance.
- Density pass on Overview: analyst-grade use of space; stat strip may act as
  chart selector (Vercel pattern); inline time-range control (Posh pattern)
  where it fits.
