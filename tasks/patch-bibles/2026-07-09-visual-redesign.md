## 2. Merge order, waves & rationale

The chart/component library, Settings, and Menu-bar are **Core-independent** (components take primitive
value types — spec §3 — and Settings/Menu-bar read existing snapshot/toggle state). Only the
Overview + Provider-Detail *analytics wiring* needs WP-1's new VM props. So:

- **Wave 1 (parallel, both start now):**
  - WP-1 builds all Core analytics + hourly + VM props + tests.
  - WP-2 builds design tokens, the entire `UI/Charts/` component library (preview-driven with sample
    value types), brand assets, and the Settings + Menu-bar redesigns — **all compiling independently**
    (no WP-1 symbols referenced yet). It also lays out Overview + Provider Detail using the components
    fed by **local preview/sample data**.
- **Merge WP-1 → dev first.**
- **Wave 2 (WP-2 continues, same worktree, rebased on updated dev):** swap the Overview + Provider-Detail
  sample feeds for the real frozen VM props (§4). This is the ONLY step that depends on WP-1. Commit,
  then merge WP-2 → dev.

Rationale: maximizes parallel wall-clock, keeps each side compiling/testing independently, and confines
the cross-package dependency to one small, well-specified wiring step against a frozen contract.

## 4. Frozen contract — `DashboardViewModel` analytics surface (WP-1 provides, WP-2 consumes)

WP-1 adds exactly these to `DashboardViewModel` (names/types are the contract; WP-2's Wave-2 wiring
targets them verbatim). All are read-only derived (computed from `snapshots` + `range` via `UsageAnalytics`):

```
// Ranged overview (all visible providers)
var range: UsageRange { get set }                         // drives everything below; default .sevenDay
var overviewTrend: [(date: Date, tokens: Int)]            // token-usage-over-time line
var providerSplit: [(provider: ProviderID, tokens: Int)]  // donut source (ranged, hidden excluded)
var overviewDelta: Double?                                // "↑18% vs yesterday" (nil if no baseline)
var streak: (current: Int, longest: Int)
var bestDay: (date: Date, tokens: Int)?
var leastActiveDay: (date: Date, tokens: Int)?
var dailyAverage: Int?

// Per provider (selected detail)
func trend(for id: ProviderID) -> [(date: Date, tokens: Int)]
func thisWeek(for id: ProviderID) -> (peakDayWeekday: Int, peakDayTokens: Int, dailyAverage: Int, delta: Double?)?
func heatmap(for id: ProviderID) -> [[Int?]]?             // 7×24, nil until hourly lands
func peakHour(for id: ProviderID) -> (hour: Int, tokens: Int)?
```

`UsageRange` (Core enum): `.sevenDay`, `.thirtyDay`, `.ninetyDay` (+ `.today` if trivial). Ranges beyond
available history clamp and the UI discloses ("only N days of data").
