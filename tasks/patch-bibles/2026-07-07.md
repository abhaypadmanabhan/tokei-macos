### WP-1 · Live quota connectors (Antigravity local RPC + Cursor plan) — Codex
- **Worktree:** `../tokei-worktrees/2026-07-07-live-quota-core`  (branch `patch/2026-07-07/live-quota-core`)
- **Scope (in):**
  - NEW `Core/Network/AntigravityQuotaClient.swift`
  - `Core/Providers/AntigravityProvider.swift` (gate + wire live fetch)
  - `Core/Network/CursorUsageClient.swift` (+`fetchStripeProfile`)
  - `Core/Providers/CursorProvider.swift` (plan label via `ProviderWarning`)
  - `Core/Models/QuotaWindow.swift` + `Core/Models/QuotaWindowType.swift` — **controlled
    append-only extension** (see below), in the same commit.
  - `Tests/ProviderTests/`, `Tests/Fixtures/` (add decode tests + literal fixtures)
- **Scope (out):** ALL of `AIUsageDashboardApp/UI/**` (incl. `SettingsView.swift`), other
  providers, `UsageProvider`/`QuotaProvider` protocol signatures, `ProviderSnapshot` field
  removals, storage. Never handle the Antigravity `ya29` token — the local path does not need it.

- **Antigravity — the endpoint (LOCAL ONLY, no ya29):**
  ```
  POST https://127.0.0.1:<httpsPort>/exa.language_server_pb.LanguageServerService/RetrieveUserQuotaSummary
  Headers: Content-Type: application/json ; x-codeium-csrf-token: <csrf>
  Body: {}
  ```
  - **Discovery** (all local, best-effort, must degrade gracefully if not found):
    - process: match `language_server --standalone` (Antigravity's bundled Go server).
    - `csrf`: the `--csrf_token <UUID>` arg (read via `sysctl(KERN_PROCARGS2)` or a `ps` read).
      Rotates each app launch. **Never log/persist/print it.**
    - `httpsPort`: the process's LISTEN ports (2 of them); the HTTPS one answers the POST with
      200 — the other throws a TLS "wrong version number". Probe and cache within the fetch.
  - **TLS:** server is self-signed → skip cert validation **scoped to 127.0.0.1 ONLY** (a
    `URLSessionDelegate` that trusts only `challenge.protectionSpace.host == "127.0.0.1"`).
    Never a global `NSAllowsArbitraryLoads` / blanket trust.
  - **Response → `QuotaWindow[]`** (real captured JSON is the test fixture; see scout doc §1):
    `response.groups[]` → `{displayName, buckets[]}`; `buckets[]` →
    `{bucketId, window:"weekly"|"5h", remainingFraction:Double 0–1, resetTime:ISO8601Z}`.
    Map each bucket to one `QuotaWindow`:
    - `type` = `.weekly` (window "weekly") or `.fiveHour` (window "5h")
    - `used` = `round((1 - remainingFraction) * 100)`, `limit` = `100`, `remaining` = `round(remainingFraction*100)`
    - `resetAt` = parse `resetTime`
    - `label` = group `displayName` (e.g. "Gemini Models")
    - `bucketKey` = `bucketId` (e.g. "gemini-weekly") — makes the `id` unique across groups
    - `confidence` = `.providerReported`, `source` = "antigravity-local-rpc"
  - Optional: `POST …/GetUserStatus` (same headers, `{}`) → `userStatus.planStatus.planInfo.planName`
    for a live plan label via `ProviderWarning` ("Plan: Pro"). Nice-to-have, not required.

- **Append-only contract extension (do exactly this, same commit):**
  - `QuotaWindowType`: append `case fiveHour` (keep existing cases + order).
  - `QuotaWindow`: append two optionals with defaults — `public let label: String?` (group name)
    and `public let bucketKey: String?` (stable unique key); add them to `init` with `= nil`
    defaults so every existing call site still compiles unchanged.
  - `QuotaWindow.id`: change to `bucketKey ?? "\(providerID.rawValue)_\(type.rawValue)"` so
    Antigravity's 4 windows get unique ids while all existing windows keep their old id.
  - This is the ONLY contract change permitted this package. It is additive/backward-compatible.

- **Cursor plan enrichment (NO gauge — plan label only):**
  - `CursorUsageClient.fetchStripeProfile()` → `GET https://api2.cursor.sh/auth/full_stripe_profile`
    with `Authorization: Bearer <JWT>` (reuse the existing JWT read + gating behind
    `cursorNetworkUsageEnabled`). Response is `text/plain` JSON; decode fields:
    `membershipType, subscriptionStatus, individualMembershipType, isYearlyPlan, isOnBillableAuto,
    customerBalance, pendingCancellationDate, lastPaymentFailed` (real sample in scout doc §2).
  - Surface a `ProviderWarning` like `Plan: Pro · monthly · auto-billing on` (compose from
    membershipType + isYearlyPlan + isOnBillableAuto). **Do NOT invent a quota gauge** — this
    account is uncapped (`maxRequestUsage:null`); honest per-model request counts stay as-is.

- **Acceptance criteria:**
  - [ ] `AntigravityQuotaClient` decodes the real captured JSON into 4 `QuotaWindow`s (2 groups
        × weekly/5h) with correct `used`/`resetAt`/`label`/`bucketKey`; unit-tested from a literal fixture.
  - [ ] `AntigravityProvider` returns the live windows ONLY when `antigravityOnlineQuotaEnabled`
        (UserDefaults) is true AND discovery succeeds; otherwise falls back to today's honest local
        output with no crash and no error surfaced to the user for the "app not running" case.
  - [ ] csrf/token never logged, persisted, or printed; TLS trust limited to 127.0.0.1.
  - [ ] `CursorProvider` shows an accurate plan label from `full_stripe_profile`; no fabricated gauge.
  - [ ] Contract extension is additive; every pre-existing call site compiles unchanged.
  - [ ] `xcodegen generate` + Core test scheme build & all tests green (85 baseline + new).
- **Test requirements:**
  - [ ] Antigravity quota decode test (literal fixture = the scout JSON) → asserts 4 windows,
        used% math, fiveHour type, unique ids, resetAt parse.
  - [ ] Cursor stripe-profile decode test (literal fixture) → asserts plan-label composition.
  - [ ] A discovery-failure test (no process) → provider returns local fallback, no throw to UI.
- **Design / UX constraints:** none (no UI in this package).
- **Known risks:**
  - App Sandbox / hardened runtime may block `ps`/`sysctl` process-arg reads and localhost
    connect → mitigation: discovery is best-effort + opt-in (default OFF); on failure, degrade
    to local data silently. Today's dev build is non-sandboxed (endpoint already proven via curl);
    sandbox/entitlements are a distribution-time follow-up (issue #8), not this package.
  - csrf rotates per app launch → never cache across fetches beyond the single call; re-discover.
  - `httpsPort` is one of two ports → probe both; cache only for the current fetch.
- **Rollback:** revert merge commit; branch retained.
