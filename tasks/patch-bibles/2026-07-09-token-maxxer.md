### WP-1 · Quota snapshot sampler + append-only series persistence (#32) — Codex
- **Worktree:** `../tokei-worktrees/2026-07-09-quota-series-sampler` (branch `patch/2026-07-09/quota-series-sampler`)
- **Scope (in):** `Core/Storage/` (new `QuotaSeriesStore.swift` + a `QuotaSample` model), a single sampling hook in `Core/Sync/SyncEngine.swift` (append one sample per sync tick per window). Tests under `Tests/`.
- **Scope (out):** `Core/Network/*` (WP-2), all `UI/*` (WP-3), `DashboardViewModel` public surface beyond wiring the sampler call, `QuotaWindow`/`ProviderSnapshot` shapes (frozen §4).
- **Files likely involved:** `Core/Storage/QuotaSeriesStore.swift` (new), `Core/Models/QuotaSample.swift` (new), `Core/Sync/SyncEngine.swift` (sampler call-site only), `Tests/QuotaSeriesStoreTests.swift` (new).
- **Acceptance criteria:**
  - [ ] Each sync appends one `QuotaSample {providerID, windowType, bucketKey, usedPercent, used, limit, resetAt, sampledAt}` per non-nil quota window to an append-only JSON store in Application Support.
  - [ ] Store is bounded (ring/retention cap — e.g. keep ≤ N days or ≤ M samples/window; document the cap; no unbounded growth).
  - [ ] Writes are atomic (temp-file + rename), crash-safe, single-writer; concurrent reads never see a torn file.
  - [ ] Dedup: identical consecutive samples (same usedPercent within a window bucket) collapse or are cheap — no spam on idle syncs.
  - [ ] Public read API: `samples(for providerID:windowType:since:) -> [QuotaSample]` (additive; used by future #41/#36 history — not consumed this patch).
- **Test requirements:**
  - [ ] Round-trip: append → reload → identical series. Retention cap enforced. Atomic-write survives a simulated mid-write (temp file left, real file intact). Fixtures = Swift string literals (§4).
- **Known risks:** unbounded file growth → mitigation: hard retention cap + test. Torn writes → atomic rename + test.
- **Rollback:** revert merge commit; branch retained.

### WP-2 · Port quota resilience layer to CursorUsageClient (#35) — Kimi K2.7
- **Worktree:** `../tokei-worktrees/2026-07-09-cursor-quota-resilience` (branch `patch/2026-07-09/cursor-quota-resilience`)
- **Scope (in):** `Core/Network/CursorUsageClient.swift` (+ a small `Core/Network/` cooldown/backoff helper if one doesn't already exist to share). Tests under `Tests/`.
- **Scope (out):** `ClaudeUsageClient.swift` (do NOT edit the source pattern — read it, mirror it), `Core/Storage/*` (WP-1), `UI/*` (WP-3), the Cursor pure parsers (`CursorUsageCSV`/`CursorUsageSummary` shapes are frozen).
- **Reference pattern:** `Core/Network/ClaudeUsageClient.swift` — mirror its error taxonomy + cooldown behaviour: `rateLimited(retryAfter:)`, `cooldownActive`, honoring `Retry-After`, backing off after a 429/rate-limit so we don't hammer Cursor and risk a lockout.
- **Files likely involved:** `Core/Network/CursorUsageClient.swift`, `Tests/CursorUsageClientTests.swift` (new/extended).
- **Acceptance criteria:**
  - [ ] `CursorUsageError` gains rate-limit/cooldown cases mirroring the Claude taxonomy (`rateLimited(retryAfter:)`, `cooldownActive`).
  - [ ] On HTTP 429 (and 5xx per the Claude pattern), the client enters a cooldown for `Retry-After` (or a sane default) and fast-fails subsequent calls until cooldown elapses — no tight retry loop.
  - [ ] Cooldown clock is injectable (a `now`/clock closure or `Date` provider) so tests are deterministic — no wall-clock sleeps in tests.
  - [ ] Existing Cursor happy-path (CSV export + usage-summary) behaviour unchanged; the cookie still leaves the process only as the `Cookie` header over TLS (no new logging of the cookie).
- **Test requirements:**
  - [ ] 429 → cooldown entered → next call fast-fails with `cooldownActive` before cooldown elapses, succeeds after (using injected clock). Happy-path regression stays green. `URLProtocol` stub or injected session — no live network.
- **Design/UX:** none (Core). If a view incidentally shows a cooldown state, that's WP-3's job — do NOT touch UI.
- **Known risks:** hammering Cursor without backoff = account lockout → this IS the mitigation. Test must not hit live cursor.com.
- **Rollback:** revert merge commit; branch retained.
