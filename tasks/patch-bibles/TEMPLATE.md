## 4. Frozen contracts (all agents, non-negotiable)

- `UsageProvider` protocol — do not change signatures.
- `UsageStore` public API — additive only.
- `ProviderSnapshot` fields — **append-only**; any new stored field MUST be added
  to `Core/Storage/ModelCodableExtensions.swift` in the SAME commit (manual Codable).
- Existing tests stay green — never delete or weaken them.
- Test fixtures = Swift string literals in `Tests/Fixtures/Fixtures.swift` (no resource files).
- No secrets outside Keychain. Raw provider responses = debug-only.
- No UI → provider direct calls (UI talks to the view model/sync engine only).

## 5. Design / UX constraints (Padzy OS "aitracker")

- ground `#131316` · surface `#1D1D22` · ink `#ECECF1` · muted `#6E6E78` · ONE accent `#FF3B70`.
- Accent = state/action ONLY (never decoration; cost values use ink, not accent).
- Mono for all numeric/data. No shadows, no gradients, no rounded card grids, radius ≤ 4px.
- Premium macOS feel, clean hierarchy. Every surface needs real empty / loading / error states.
- UI is LOCKED to a dedicated Claude Code UI agent scoped to `AIUsageDashboardApp/UI/`; it
  runs the `padzy-os` skill (aitracker theme). UI is never baked into a core package and
  never assigned to a non-Claude agent. Tokens above are the reference/guard if a non-UI
  agent incidentally touches a view — external agents never "run the skill."

## 7. Global rollback plan

- Each work package merges to `dev` as a single `--no-ff` commit → revert is one `git revert -m 1 <sha>`.
- `dev` is disposable; `main` is the safety net. Nothing reaches `main` without `/dev-approved`.
- Worktrees and `patch/*` branches are retained until the day's patch ships, then pruned.
