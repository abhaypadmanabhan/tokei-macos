# AI Usage Dashboard

A local-first native macOS app for tracking AI coding tool usage across multiple providers.

## Supported Providers (MVP)

- Anthropic Claude Code
- OpenAI Codex
- Cursor
- Cline / Cline Pass
- Antigravity

## Project Structure

```
AIUsageDashboard/
├── AIUsageDashboard.xcodeproj   # Generated by xcodegen
├── project.yml                  # xcodegen specification
├── AIUsageDashboardApp/
│   ├── App/                     # @main app entry point
│   ├── UI/                      # SwiftUI views (Dashboard, MenuBar, ProviderDetail, Settings, Components)
│   ├── Core/                    # Shared core framework (AIUsageDashboardCore)
│   │   ├── Models/              # Provider, Quota, Token, Cost models
│   │   ├── Providers/           # Provider protocols and adapters
│   │   ├── Parsing/             # Claude JSONL parser
│   │   ├── Sync/                # SyncEngine, DashboardViewModel
│   │   ├── Storage/             # SwiftData/SQLite persistence skeleton
│   │   ├── Notifications/       # Notification engine skeleton
│   │   ├── Security/            # Keychain wrapper
│   │   └── Utilities/           # Date helpers
│   └── Resources/               # Assets and Info.plist
├── AIUsageDashboardWidgets/     # WidgetKit source files (not yet in Xcode project)
├── Tests/                       # Unit tests and JSONL fixtures
└── docs/                        # Engineering docs
```

## Build & Run

Requires macOS 14+ and Xcode 15+.

1. Open `AIUsageDashboard/AIUsageDashboard.xcodeproj` in Xcode, or build from the command line:

```bash
cd AIUsageDashboard
xcodebuild -project AIUsageDashboard.xcodeproj -scheme AIUsageDashboardApp -destination 'platform=macOS' build
```

2. To regenerate the Xcode project from `project.yml`:

```bash
cd AIUsageDashboard
xcodegen generate
```

3. To run tests:

```bash
cd AIUsageDashboard
xcodebuild -project AIUsageDashboard.xcodeproj -scheme AIUsageDashboardCore -destination 'platform=macOS' test
```

## What Works

- Native macOS SwiftUI app shell with a dashboard and menu bar extra.
- Core framework with provider model, quota windows, token usage, and confidence labels.
- Claude Code provider adapter that discovers local JSONL logs and parses them with streaming.
- Skeleton providers for Codex, Cursor, Cline, and Antigravity.
- `SyncEngine` and `DashboardViewModel` wiring.
- Persistence, notifications, and security skeletons.
- Unit tests for the core models and Claude JSONL parser.

## What Is Stubbed

- Codex, Cursor, Cline, and Antigravity providers return `unavailable` metrics and only log detection is partially implemented.
- SwiftData entities and historical persistence are not fully implemented.
- FSEvents file watcher is not implemented.
- Notification threshold engine is not implemented.
- WidgetKit extension source files exist but are not yet included in the generated Xcode project.
- App icons and visual polish are not present.
